# Dr.Tools UI 设计规范

> 版本：v1.0  
> 基线来源：`src/App.vue`、`src/layouts/SidebarNav.vue`、`src/modules/**`、`src/theme/appearance.ts`、`src/styles.css`

## 1. 设计目标

本规范用于约束 Dr.Tools 当前桌面应用的视觉与交互基线，保证：

- 桌面端界面清晰、稳定、可扫描
- 深色 / 浅色主题行为一致
- `Liquid Glass` 效果边界明确，不侵入基础配色
- 模块页在统一壳层下保持一致的布局语言

## 2. 视觉原则

- 以中性色为基础，强调色只用于当前态、可操作态、焦点态
- 层级主要通过背景、边框、字号和间距建立，不依赖重色块
- 卡片、列表、日志区保持统一描边与圆角体系
- 滚动区域默认隐藏滚动条，维持桌面应用的干净感
- 输入框、按钮、菜单选中态必须服从当前主题和颜色配置

## 3. 主题系统

### 3.1 主题模式

- `dark` / `light` / `auto`
- 主题状态挂载到 `document.documentElement.dataset.theme`

### 3.2 Liquid Glass

- `Liquid Glass` 只控制右侧顶部标题栏的背景表现
- `透明`：使用当前主题颜色的半透明版本，并保留 `backdrop-filter` 玻璃效果
- `色调`：使用当前主题对应的纯色背景，不使用半透明玻璃背景
- `Liquid Glass` 不负责修改窗口背景、卡片背景、输入框背景、菜单背景等基础色值
- 所有内容卡片统一无阴影
- 菜单容器阴影不属于 `Liquid Glass` 规则

### 3.3 强调色

- 菜单 list 按钮选中态：背景使用当前配置中的纯色，文字自动使用反色
- 图形颜色跟随当前主题模式
- `透明` 表现为当前主题颜色的透明版本
- `色调` 表现为当前主题颜色的纯色版本

## 4. 颜色基线

### 4.1 深色模式

- 窗口背景色：`#2d2d2c`
- 顶部标题栏背景色：`#2d2d2c`
- 卡片容器背景色：`#333332`
- 菜单容器背景色：`#232322`
- 输入框背景色：`#353535`
- 主文字颜色：`#ffffff`
- 提示文字颜色：`#a2a2a2`
- 容器边框颜色：`#444444`
- 分割线颜色：`#3d3d3c`

### 4.2 浅色模式

- 窗口背景色：`#ffffff`
- 顶部标题栏背景色：`#ffffff`
- 卡片容器背景色：`#f7f7f7`
- 输入框背景色：跟随浅色输入控件体系
- 侧栏需要轻阴影，但阴影尺寸必须克制
- 图形颜色按浅色主题重新计算，保证可识别性

## 5. 布局规范

### 5.1 主壳布局

- 左侧固定导航，右侧为内容区域
- 顶部使用通用标题栏，不在页面内容区重复写标题 + 副标题
- 设置页入口固定在一级菜单底部
- 内容区超出时允许滚动，但滚动条默认隐藏

### 5.2 页面结构

- 下载、任务记录、设置、直播录制都沿用统一内容布局基线
- 业务页面优先使用清晰的单列或稳定双列，不做碎片化卡片拼贴
- 表单区、日志区、结果区按信息优先级排序，不做无意义装饰性分栏

### 5.3 任务记录

- 每次提交按批次记录
- 查看批次后再展示该批次下的子任务
- 明细查看使用新窗口展示
- 空状态统一使用 `src/assets/icons/empty.svg`

## 6. 组件规范

### 6.1 Surface

- 统一用于卡片容器、日志容器、设置块、列表块
- 基础样式：背景色、边框、圆角服从主题 token
- 所有内容卡片统一不使用阴影

### 6.2 输入控件

- 不随意覆盖系统 `select` 的尺寸与复杂样式
- 下拉组件只做颜色兼容，不强加宽度与多余装饰
- `input / textarea / select` 的背景、边框、文字颜色必须适配明暗主题

### 6.3 按钮

- 主按钮圆角保持克制，不使用过度胶囊化样式
- 次按钮、危险按钮、轻操作按钮使用统一语义色
- 焦点反馈统一使用主题 focus ring

### 6.4 日志与列表

- 处理日志按单条任务展示，最新优先
- 列表项需要稳定的垂直居中与统一间距
- 平台、附加内容等选项型列表需要保持规则网格布局

## 7. 交互规则

- 路由导航由窗口顶部返回 / 前进能力配合页面路由共同工作
- 页面内部不重复制造第二套导航体系
- 提交类页面反馈应直接、明确，避免出现 `[object Object]` 这类原始错误暴露
- 下载完成通知按批次触发，而不是按单条任务逐条弹出

## 8. 开发约束

- 不在组件内部硬编码与主题冲突的颜色
- 新样式优先沉淀为可复用 token 或现有语义类
- 文案、图标、空状态、选中态必须同时验证深色与浅色模式
- 任何新页面都必须先服从现有壳层、主题、间距和组件基线
