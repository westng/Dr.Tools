# Dr.Tools 设计系统

## 1. 文档定位

本文档用于沉淀 `Dr.Tools` 当前已经落地的设计规则，作为后续界面设计、前端实现和视觉验收的统一依据。

约束原则：

- 以当前代码实现为准，不写脱离代码的理想化规范
- 先统一主题令牌，再扩展组件样式
- 普通页面不重复堆叠标题和副标题，主标题统一由顶部标题栏提供
- 所有新增页面必须同时兼容浅色模式与深色模式

## 2. 设计原则

- 简洁：减少无意义装饰，优先保证信息清晰
- 一致：颜色、字号、圆角、间距统一复用全局规则
- 克制：卡片、列表、表单不叠加过多视觉层次
- 实用：优先保证录入效率、列表可读性和状态识别
- 可维护：优先使用全局 CSS 变量，不散落硬编码样式

## 3. 主题系统

项目当前支持两层主题能力：

- 主题模式：`light` / `dark`
- 标题栏风格：`Liquid Glass`
  - `transparent`：半透明玻璃效果
  - `tinted`：纯色主题背景

### 3.1 Liquid Glass 规则

`Liquid Glass` 只控制顶部标题栏 `content-toolbar` 的背景表现，不控制卡片阴影，不控制页面卡片层次。

规则如下：

- `transparent`
  - 标题栏使用半透明背景
  - 启用 `backdrop-filter: blur(16px) saturate(130%)`
  - 浅色模式下基于白色透明化，深色模式下基于当前窗口背景透明化
- `tinted`
  - 标题栏直接使用当前主题背景色
  - 不启用玻璃模糊效果

明确约束：

- 所有卡片默认无阴影
- 只有菜单侧栏在浅色模式下允许使用轻微阴影
- 不能把 `Liquid Glass` 扩展到卡片、表格、设置块等普通容器

## 4.3 颜色使用规则

- 窗口底色统一使用 `--bg-main`
- 普通卡片统一使用 `--bg-card`
- 菜单与导航容器统一使用 `--sidebar-bg` 或 `--bg-panel`
- 输入框、下拉框、文本域统一使用 `--bg-input`
- 主文案统一使用 `--text-main`
- 说明性文案统一使用 `--text-muted`
- 边框统一使用 `--stroke-soft`
- 分割线统一使用 `--divider-soft`
- 选中、激活、聚焦统一围绕 `--accent`

## 5. 字体系统

## 5.1 字体族

全局字体栈：

```css
font-family: 'SF Pro Text', 'PingFang SC', 'Helvetica Neue', sans-serif;
```

规则：

- 保持系统级、桌面工具型字体观感
- 中文优先保证 `PingFang SC` 的稳定表现
- 不额外引入风格化字体

## 5.2 字号层级

| 场景 | 字号 | 说明 |
| --- | --- | --- |
| 全局基础字号 | `14px` | 默认正文基线 |
| 顶部标题栏标题 | `30px` | `toolbar-title` |
| 页面标题 | `20px` | 仅保留给少量独立页面或局部标题 |
| 卡片标题 | `18px` | 模块标题 |
| 主操作文字 | `15px` | 主按钮 / 关键操作 |
| 常规标签 / 次级信息 | `12px` | 标签、状态、辅助信息 |
| 紧凑日志 / 超小元信息 | `11px` | 紧凑型日志内容 |

当前约束：

- 页面默认不再重复渲染“标题 + 副标题”区块
- 常规页面标题由顶部标题栏统一承载
- 内容区字号以 `14px` 为主，不随意放大

## 5.3 字重与行高

- 常规正文：`400`
- 重点标题：`600` ~ `700`
- 全局基础行高：`1.45`
- 标题行高：`1.0` ~ `1.2`
- 日志与元信息行高：`1.4`

## 6. 圆角、边框与阴影

## 6.1 圆角

| 场景 | 圆角 |
| --- | --- |
| 侧栏容器 | `16px` |
| 普通卡片 | `12px` |
| 默认按钮 | `10px` |
| 输入框 / 文本域 | `8px` |
| 胶囊标签 / 状态标签 | `999px` |
| 虚线添加入口 | `14px` |

## 6.2 边框

- 默认边框：`1px solid var(--stroke-soft)`
- 侧栏边框：`2px solid var(--sidebar-stroke)`
- 分隔线：`1px solid var(--divider-soft)`

## 6.3 阴影

- 普通卡片：默认无阴影
- 菜单侧栏：
  - 深色模式：无阴影
  - 浅色模式：轻微阴影，保持分层即可
- 阴影强度要小，不能抢夺内容视觉优先级

## 7. 间距系统

当前实现主要围绕以下间距节奏：

- `4px`
- `6px`
- `8px`
- `10px`
- `12px`
- `14px`
- `16px`
- `18px`
- `20px`

使用规则：

- 控件内部小间距优先 `4 / 6 / 8`
- 表单与列表内部优先 `8 / 10 / 12`
- 卡片内边距优先 `12 / 16`
- 页面级留白优先 `16 / 20`

## 8. 布局系统

## 8.1 主窗口布局

主窗口采用双栏布局：

- 左侧菜单栏：固定 `280px`
- 右侧内容区：`minmax(0, 1fr)`

```css
grid-template-columns: 280px minmax(0, 1fr);
```

## 8.2 标题栏

标题栏规则：

- 最小高度：`72px`
- 位置：内容区顶部
- 支持路由后退 / 前进
- 背景由主题与 `Liquid Glass` 联合决定

标题栏中只保留必要元素：

- 导航按钮
- 当前页面标题

不再在页面主体内重复添加相同标题和副标题。

## 8.3 内容区

- 内容区默认纵向布局
- 页面主体区域允许滚动
- 滚动条默认隐藏
- 独立窗口统一使用 `standalone-shell`

## 8.4 独立窗口

适用于添加账号、编辑信息、查看明细等子窗口。

规则：

- 继承全局主题变量
- 继承深色 / 浅色模式
- 继承标题栏之外的基础样式规则
- 底部操作区允许固定浮动，但内容本身必须能完整滚动

## 9. 基础组件规范

## 9.1 卡片 `surface`

基础规则：

- 背景：`var(--bg-card)`
- 圆角：`12px`
- 默认内边距：`12px`
- 默认无阴影

补充约束：

- `settings-block` 在浅色模式固定为 `#f7f7f7`
- `settings-block` 在深色模式固定为 `#333332`
- 卡片不跟随 `Liquid Glass` 变化

## 9.2 菜单侧栏 `sidebar`

基础规则：

- 圆角：`16px`
- 边框：`2px solid var(--sidebar-stroke)`
- 背景：`var(--sidebar-bg)`
- 深色模式无阴影
- 浅色模式允许轻阴影

导航项规则：

- 导航项背景默认使用 `--bg-card`
- 激活项使用 `--accent`
- 激活项文字使用 `--accent-contrast`

## 9.3 按钮

基础规则：

- 背景：`var(--bg-input)`
- 边框：`1px solid var(--stroke-soft)`
- 圆角：`10px`
- 内边距：`8px 12px`
- Hover：轻微与强调色混合

约束：

- 禁用态统一降低透明度
- 不使用大圆角主按钮
- 主按钮保持克制，不做夸张渐变和大面积阴影

## 9.4 输入框 / 文本域 / 选择器

基础规则：

- 背景：`var(--bg-input)`
- 边框：`1px solid var(--stroke-soft)`
- 圆角：`8px`
- 内边距：`8px 10px`
- 字号：沿用全局 `14px`

约束：

- 下拉组件优先保留系统默认结构与宽度
- 只做颜色兼容，不追加多余视觉样式
- 深色模式下输入框背景固定为 `#353535`

## 9.5 标签与状态

基础规则：

- 胶囊圆角：`999px`
- 字号：`12px`
- 内边距：`0 10px`

状态建议：

- 默认：中性灰
- 监控中：强调色弱填充
- 直播中：高识别暖色提示
- 成功 / 完成：绿色提示
- 异常 / 失败：红色提示

## 9.6 表格与列表

规则：

- 表头与表体必须严格对齐
- 超出内容默认单行省略，不换行
- 操作列优先保证可读性和可点击性
- 分页固定在底部区域时，要和内容视觉分层明确
- 分页容器可根据 `Liquid Glass` 决定是否使用透明玻璃效果

## 9.7 空状态

规则：

- 优先使用统一空状态图形
- 当前空状态图路径：`src/assets/icons/empty.svg`
- 说明文案使用 `--text-muted`
- 不叠加冗长描述

## 10. 图形适配规则

- 图形颜色由当前主题模式决定
- “透明” 表现为当前主题色的透明版本
- “色调” 表现为当前主题色的不透明版本
- 浅色模式下透明图形必须保证可识别度，不能因为透明度过低而看不清
- 深色与浅色模式下图形都必须优先保证对比度

## 11. 页面类型约束

## 11.1 设置页

- 以设置块卡片组织内容
- 不保留额外导入导出型占位区块
- 下载路径等关键配置使用明确命名，不使用模糊描述

## 11.2 视频下载页

- 输入链接区优先突出录入效率
- 平台、附加内容采用紧凑型列表组织
- 处理日志按单条任务日志展示，按最新排序
- 不制造过多并列卡片造成割裂

## 11.3 直播录制页

- 核心对象是录制账号，不是单条日志
- 录制账号卡片优先展示头像、名称、UID、状态和操作
- 添加账号入口保持明确、直接
- 新建 / 编辑 / 查看明细走独立窗口

## 11.4 任务记录页

- 按批次维度展示，不按单个子任务平铺主视图
- 列表排版优先保证表头对齐、内容省略和操作可读性
- 刷新、分页等辅助操作要和主列表有明确层级区分

## 12. 实现约束

- 优先使用 `src/styles.css` 中的主题令牌
- 新增颜色先抽象为变量，再落到组件
- 新增页面先复用现有布局骨架，再做模块扩展
- 如果实现与本文档冲突，应先统一规则，再修改代码
- 设计系统文档必须跟着实现更新，不能长期失真
